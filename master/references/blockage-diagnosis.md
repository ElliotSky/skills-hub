# 卡点诊断手册

本文件定义 Master skill 在教学中识别和突破卡点的操作协议。当学生无法继续推进时，使用本手册诊断原因并选择精准干预。

---

## 一、卡点四类型

在开始重复追问之前，先诊断卡点类型——不同类型需要完全不同的干预方式。

| 类型 | 定义 | 识别信号 |
|---|---|---|
| **A. 概念理解卡** | 学生不理解当前概念本身的含义 | 解释绕圈、换词复述原句、用定义解释定义 |
| **B. 推理链断裂** | 理解各个点，但连不上 | 能说 A 能说 C，但说不出 A→B→C 的逻辑 |
| **C. 前置知识缺失** | 当前概念依赖学生不具备的前置知识 | 正确回答前一轮，但此轮答案与上下文完全脱节 |
| **D. 语言表达卡** | 理解了但说不出来 | 说"我懂了但不知道怎么描述"、用手势或类比暗示但无法展开 |

---

## 二、诊断动作（优先于追问）

**当学生连续 2 轮没有推进**（不是答错，而是原地踏步），先诊断，再干预。

### 诊断问题（只问其中 1 个）

针对**A. 概念理解卡**：
> 「不用解释这个概念，你能给我一个你认为符合它的具体例子吗？」（如果连例子也给不了 → 确认是 A 型卡点）

针对**B. 推理链断裂**：
> 「我们已经知道了 X 和 Z，你觉得中间缺了什么步骤？」（如果说不出中间步骤 → 确认是 B 型卡点）

针对**C. 前置知识缺失**：
> 「这个问题用到了 [某个前置概念]，你之前接触过吗？能说说你对它的理解吗？」（如果描述偏差很大 → 确认是 C 型卡点）

针对**D. 语言表达卡**：
> 「你不用完整解释，先用最不准确的话说一遍，我们再一起打磨。」（如果学生能说出一个方向 → 确认是 D 型卡点）

---

## 三、精准干预策略

### A 型（概念理解卡）→ 降维 + 换角度

不要重复解释同一个概念，而是：
1. **退到更具体的情境**：给一个极端简化的特例，让概念"看得见"
2. **换角度进入**：如果从"是什么"进不去，改从"它能做什么做不到什么"进入
3. **对立案例**：「我现在给你一个不属于这个概念的例子，你能说说为什么它不符合吗？」

禁止：再给一遍定义、再换一个类比（若第一个类比没起效，第二个类比也大概率失效）

### B 型（推理链断裂）→ 显式补链

不要让学生猜缺失的步骤，而是：
1. **逐步列出已确认的节点**：「你已经知道 A，也知道 C，我们来看看 B 会是什么」
2. **只补一个中间步骤**，然后立刻问「有了这个，你能接到 C 了吗？」
3. **不补全推理链**——目的是让学生自己走完最后一段

禁止：一次性把完整推理链补满，导致学生没有推导过程的参与

### C 型（前置知识缺失）→ 快速补丁

时间优先，不走完整流程，而是：
1. 确认缺失的前置概念是什么（精准到一个具体概念，不是一个主题）
2. 用 1-2 个问题引导学生自己推出这个前置概念（如果 3 问内推不出来 → 直接给出）
3. 给出后立即问「有了这个，你能回头试试刚才的问题吗？」
4. 在 `progress.md` 的「前置知识补丁」列表记录：被动补充的前置概念 + 所属卡点上下文，待后续系统补全

禁止：进入完整的前置阶段教学（偏题太远，打断节奏）

### D 型（语言表达卡）→ 共同建构

学生理解了但说不出，这不是真正的卡点，而是语言化能力的障碍：
1. 给学生一个不完整的句子让他填空：「这个概念说的是：当 [某条件] 时，[学生来填]」
2. 接受学生给出的任何语言版本，追问「我听到你说的是 X，这句话的意思是……？」帮他打磨
3. 学生打磨出一个说法后，立刻要求他"再说一遍，这次用自己的话"

禁止：替学生说出完整解释（剥夺他建构语言的机会）

## 四、快速突破 vs 回溯决策树

```
学生无法推进
       │
       ▼
连续 2 轮原地踏步？
  ├── 否 → 继续当前追问策略
  └── 是 → 诊断卡点类型（问诊断问题）
              │
              ▼
         诊断结果？
          ├── A 型 → 降维 + 换角度
          ├── B 型 → 显式补链（只补一个节点）
          ├── D 型 → 共同建构（填空法）
          └── C 型 → 前置知识缺失
                       │
                       ▼
              缺失的前置概念是否是本主题学过的？
               ├── 是 → 快速补丁（重新引导）→ 回到原问题
               │        ├── 成功 → 继续
               │        └── 仍无法推进 → 记录"搁置卡点"（前置概念未真正掌握）
               └── 否（从未涉及）→
                       │
                       ▼
                  3 问内能引导学生推出？
                   ├── 是 → 继续（不打断流程）
                   └── 否 → 直接给出 → 记录"前置知识补丁"（progress.md §前置知识补丁）
                             → 完成本阶段后，单独开一段前置知识补全
```

---

## 五、卡点次数上限

同一个卡点：
- **A/B/D 型**：精准干预后，如果 2 轮内仍无进展 → 标记为「阶段性搁置」，先跳过这个概念继续后面的内容，在 `progress.md` 记录「搁置卡点」，下一阶段开场时重新回来
- **C 型**：快速补丁后，如果学生仍无法推进 → 升级为「前置知识专题补全」，视为一个独立的学习任务

**永远不要在同一个卡点上无限重复追问**——次数上限就是触发机制改变的信号。
