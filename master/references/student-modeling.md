# 学生建模手册

本文件定义学生画像的六维结构、节奏判据、更新规则，供 Master skill 在教学过程中维护 `student-profile.md`。

## 一、学生画像六维（硬锁，禁止新增顶层字段）

### 字段总览

| 维度 | 作用 | 取值 / 标签类型 |
|---|---|---|
| 认知风格 | 学生更容易从哪个角度切入新概念 | 具象优先 / 抽象优先 / 类比驱动 |
| 节奏档位 | 学生当前的吸收速度 | slow / standard / fast |
| 动机与目标 | 为什么学、要学到什么程度 | 自由文本 + 深度标记（了解 / 应用 / 研究） |
| 先验地图 | 已掌握的相关知识 + 误概念 | 两个子列表：已会 / 误概念 |
| 薄弱信号 | 观察到的具体弱点标签 | 标签列表（见下） |
| 元认知水平 | 学生是否能准确感知自己的不懂之处 | 高 / 中 / 低 |

## 一·五、认知风格→教学动作映射

知道学生的认知风格后，Master 必须相应调整教学策略，而不只是记录标签。

| 认知风格 | Master 的对应调整 |
|---|---|
| 具象优先 | 先给具体实例再归纳抽象；提问从「给我一个例子」而非「解释一下」切入；定义出现之前必须先有可操作的情境 |
| 抽象优先 | 可直接给结构化定义，但紧跟「你能给我一个验证这个定义的例子吗？」强制例子检验；不必绕路具体情境 |
| 类比驱动 | 主动提供跨域类比（「这和 X 领域的 Y 是不是类似？」），但立刻追问「这个类比在哪里会失效？」引导学生发现边界；避免类比成为掩盖理解漏洞的捷径 |

认知风格可以在不同主题上有所不同，每次新主题开启时重新观察 3-5 轮再判断。
- `术语混淆`：用词不精确，混用近似概念
- `跳步`：推导中略过关键步骤，意识不到断链
- `迁移弱`：能记住结论但不能在新场景中应用
- `定义依赖`：只会背定义，拆掉名词就不会推
- `例子依赖`：只能在特定例子语境下理解，无法抽象
- `元认知低`：不知道自己不懂在哪，或者误以为自己懂了

## 二、`student-profile.md` 写法规则

### 结构

```markdown
---
name: student-profile
type: user
description: 跨主题学生画像，六维认知档案
---

## 认知风格
**当前判断**：[具象优先 / 抽象优先 / 类比驱动]
**Evidence**：
- [YYYY-MM-DD topic-slug stage-NN] 观察描述（一句话）
- ...

## 节奏档位
**当前档位**：[slow / standard / fast]
**Evidence**：
- [YYYY-MM-DD topic-slug stage-NN] 观察描述
- ...

## 动机与目标
**深度**：[了解 / 应用 / 研究]
**内容**：[自由文本，描述为什么学 + 想达到什么程度]
**Evidence**：
- [YYYY-MM-DD topic-slug stage-NN] 观察描述
- ...

## 先验地图
**已会**：
- 知识点 A（自评 / 观察到的能力）
- ...
**误概念**：
- 误以为 X，实际上 Y [YYYY-MM-DD topic-slug stage-NN]
- ...

## 薄弱信号
**当前标签**：[标签列表]
**Evidence**：
- [YYYY-MM-DD topic-slug stage-NN] `标签名` — 具体表现
- ...

## 元认知水平
**当前判断**：[高 / 中 / 低]
**Evidence**：
- [YYYY-MM-DD topic-slug stage-NN] 观察描述
- ...
```

### 写法铁律
1. **只能追加**：永远不修改已有的 evidence 条目，只在末尾添加新条目
2. **必须带戳**：每条 evidence 必须有 `[YYYY-MM-DD topic-slug stage-NN]` 前缀
3. **禁止扩顶层字段**：如果发现某个观察放不进六维，想想放哪个维度最合适，而不是新增维度
4. **「当前判断」字段**：根据最新 evidence 更新，旧的 evidence 保留作历史

## 三、节奏判据（可操作信号）

### Fast 信号（学生理解超过预期）
- 一次提问就抓住了追问的核心，不需要引导
- 主动提出比当前层次更深的问题
- 自发类比：「这和我之前学过的 X 是不是类似？」
- 用自己的话精炼，而不是复述原词
- 在 Master 提问之前已经想到边界情形或反例

### Slow 信号（学生需要更多支撑）
- 费曼复述时丢失关键要素（只记住名词，忘了关系）
- 举的例子与定义不自洽（以为理解了但例子是错的）
- 在同一个概念层次反复提问，而不是往深处走
- 对 Master 的追问沉默超过明显时间（文字对话中多次「嗯」/「啊」或转移话题）**⚠️ 注意：若沉默后学生给出高质量回答（能自发展开、类比、或提出更深问题），本次沉默不计入 Slow 信号，应视为「思考型沉默」并追加一条 Fast 信号**
- 使用「大概是」「感觉好像」等不确定语气而无法进一步说明

### 调档规则（防止抖动）
- **连续 2 个同方向信号**才更新节奏档位
- **单次强信号**（如学生直接说「你讲慢点」或「太简单了，加快」）→ 立即调档
- 调档后观察 2-3 轮确认，避免来回震荡

### 多主题节奏独立性
- 节奏档位**按主题独立**：同一个学生可以在数学上是 slow，在文学上是 fast
- 学生画像中记录的是「当前最活跃主题的节奏」，切换主题时需重新评估

## 三·五、微观变速机制（子概念级别的加速与跳过）

节奏档位是**阶段级**的粗粒度控制；微观变速是**子概念级**的细粒度控制，独立于档位判断。

### 微观加速（当学生在某个子概念上已经通透）

**触发条件（满足其中一条即可）**：
- 学生连续 2 次从不同角度正确回答同一子概念的问题
- 学生主动提出了该子概念的边界情形或反例（说明理解深度已超预期）
- 学生在 Master 问之前已经推出了 Master 打算问的下一个问题

**执行动作**：
1. 不再追问该子概念，立即跳到下一个驱动问题（跨越中间层次）
2. 在阶段文件 `stages/stage-NN.md` 中将已加速跳过的驱动问题标注 `[micro-accelerated]`

### 关键路径标记（在 stages/*.md 中使用）

阶段知识图谱中的驱动问题分两类：

| 标记 | 含义 | 可跳过？ |
|---|---|---|
| `[核心]` / `[入门]` 无标注 | 必须在此停留，是后续概念的前置 | 否 |
| `[可压缩]` | 如果学生已有相关背景，可快速通过 | 是（需确认学生已有基础）|
| `[fast-only]` | 仅对 fast 档位学生展开 | 是（slow/standard 可跳过）|

### 标记判断标准

在创建 stage 文件时，根据以下标准判断：

| 标记 | 何时使用 | 判断依据 |
|---|---|---|
| 无标注（核心） | 后续概念强依赖，必须停留 | 如果跳过，后续概念无法理解 |
| `[可压缩]` | 学生可能有相关背景 | 该概念在其他领域常见，或属于通识知识 |
| `[fast-only]` | 扩展内容，非核心路径 | 深化理解但不影响后续推进 |

**标记原则**：默认不标记（即核心）。只有明确属于"可压缩"或"fast-only"时才标注。

### 跳过判据

以下情形可以跳过某个驱动问题（不等待信号，直接决策）：
- 学生在入学提问中已明确提到该概念作为先验知识
- 学生在上一个问题的回答中已经自发触及该问题的核心
- 该问题被标注为 `[可压缩]` 且学生节奏档位为 fast

**跳过时不要沉默跳过**，要说一句：「这个点你刚才已经触到了，我们直接往前走。」

## 四、更新触发点

| 触发时机 | 更新内容 | 操作 |
|---|---|---|
| 每 3-5 轮对话后 | 节奏信号 / 薄弱信号的新观察 | 向对应维度追加一条 evidence |
| 每个阶段收尾 | 强制全维度检查，更新「当前判断」字段 | 读画像 → 逐维度确认或修正「当前判断」→ 追加新 evidence |
| 学生明确反馈 | 节奏（「太快了」/ 「能难一点吗」）或困难点 | 立即追加，并更新当前档位 |
| 新主题开启时 | 节奏档位重置为 adaptive，先观察 3-5 轮再判断 | 追加一条「切换主题，重新观察节奏」的 evidence |
| 误概念被纠正时 | 从「误概念」列表标注已纠正 | 在原条目后加 `✓ [YYYY-MM-DD]` |
| 学生主动说「我之前对 X 的理解是错的」| 立即更新先验地图 | 在误概念列表原条目加 `✓ 自行纠正 [YYYY-MM-DD]`；若该误概念尚未记录，先补记再标注纠正 |
| 学生出现 Fast 信号但未达连续 2 个阈值 | 预备标注，不正式调档 | 在节奏档位 evidence 加「准 Fast [YYYY-MM-DD]」标注，等待下一个信号确认 |

## 五、初始建档流程

当 `student-profile.md` 不存在时（第一次开启新主题）：

1. 通过入学提问收集基础信息（目标、背景、约束）
2. 写画像骨架（所有维度均为空 evidence，「当前判断」设为「待观察」）
3. 在 `MEMORY.md` 中挂索引：`- [学生画像](student-profile.md) — 跨主题认知档案，Master skill 教学核心依据`
4. 开始教学，用前 3-5 轮对话初始化首批 evidence
