# 锚点探测与验证

## 核心原理

**公理**：学习 = 在已有认知结构中整合新信息

**推论**：
1. 没有"锚点"（已有认知），新信息无法挂载
2. 学生可能**误以为自己懂**（Dunning-Kruger 效应）
3. 必须验证锚点的真实性，而非仅依赖学生自述

---

## 锚点的四种状态

| 状态 | 定义 | 验证标准 | 教学策略 |
|------|------|---------|---------|
| **真懂** | 能准确解释 + 能举例 + 能识别边界 | 复述准确 + 举例恰当 + 能回答"什么情况下不成立" | 可作为锚点，直接挂载新概念 |
| **半懂** | 能说出定义，但不能举例或应用 | 复述准确，但举例失败或答不出应用场景 | 需先加固（补充例子 + 应用），再作为锚点 |
| **误概念** | 解释错误，或理解有偏差 | 复述错误，或举例暴露误解 | 必须先拆除误概念，再重建 |
| **不懂** | 承认不知道，或完全答不上来 | 无法复述 | 标记为"需学习"，不能作为锚点 |

---

## 验证流程

### 步骤 1：收集学生声称的"已知"

在入学提问中，问：
```
"在学这个主题之前，你已经知道哪些相关知识？"
```

学生可能回答：
- "我学过线性代数，知道矩阵和向量"
- "我了解一点微积分，但不太熟"
- "我完全没接触过这个领域"

### 步骤 2：从声称中提取概念列表

```python
声称的已知概念 = ["矩阵", "向量", "微积分"]
```

### 步骤 3：逐个验证（抽样）

**原则**：
- 如果学生声称的概念 ≤ 3 个 → 全部验证
- 如果学生声称的概念 > 3 个 → 抽样验证（选择最关键的 3-4 个）

**验证问题模板**：

```
"用你自己的话解释一下 [概念] 是什么？"

如果学生能复述：
  "能举个例子吗？"
  
如果学生能举例：
  "在什么情况下 [概念] 不适用？"（测试边界理解）
```

### 步骤 4：根据回答分类

**真懂示例**：
```
导师："用你自己的话解释一下向量是什么？"
学生："向量是有大小和方向的量，比如力、速度都是向量。"
导师："能举个例子吗？"
学生："比如一个物体受到向右 5N 的力，可以表示为 (5, 0)。"
导师："在什么情况下向量不适用？"
学生："如果只关心大小不关心方向，比如温度、质量，就不能用向量。"

→ 分类：真懂
```

**半懂示例**：
```
导师："用你自己的话解释一下矩阵是什么？"
学生："矩阵就是一个数字的表格。"
导师："能举个例子吗？"
学生："呃……就是一堆数字排成行和列？"
导师："矩阵可以用来做什么？"
学生："不太清楚……"

→ 分类：半懂（知道定义，但不知道用途）
```

**误概念示例**：
```
导师："用你自己的话解释一下导数是什么？"
学生："导数就是函数的斜率。"
导师："能举个例子吗？"
学生："比如 y = x^2 的导数是 2x，所以斜率是 2x。"
导师："那在 x = 1 这个点，斜率是多少？"
学生："2。"
导师："如果我问你 y = x^2 在区间 [0, 2] 上的斜率是多少？"
学生："呃……平均斜率？"

→ 分类：误概念（混淆了"导数"和"斜率"，没理解导数是局部性质）
```

**不懂示例**：
```
导师："用你自己的话解释一下特征值是什么？"
学生："不好意思，我没学过这个。"

→ 分类：不懂
```

### 步骤 5：生成真实的先验地图

写入 `student-profile.md`：

```yaml
prior_knowledge:
  verified_anchors:  # 真懂，可作为锚点
    - concept: "向量"
      verified_at: "2026-04-22"
      evidence: "能准确解释定义、举例、识别边界"
      
  needs_reinforcement:  # 半懂，需加固
    - concept: "矩阵"
      verified_at: "2026-04-22"
      issue: "知道定义，但不知道用途和应用场景"
      
  misconceptions:  # 误概念，需拆除
    - concept: "导数"
      verified_at: "2026-04-22"
      issue: "混淆了导数和斜率，没理解导数是局部性质"
      
  unknown:  # 不懂，需学习
    - concept: "特征值"
```

## 认知负荷上限探测

### 目标

测量学生的"概念吞吐率"，即单位时间内能理解多少复杂度的概念。

### 流程

**步骤 1：选择测试概念**

选择一个"中等复杂度"的概念（复杂度权重 = 3），且与主题相关。

例如：
- 学线性代数 → 测试"线性组合"
- 学微积分 → 测试"极限的直观理解"
- 学概率论 → 测试"条件概率"

**步骤 2：传递概念**

用 `guided` 模式传递（给框架，让学生填空）：

```
"我们来快速过一个概念，看看你的理解速度。

线性组合的意思是：用几个向量，乘以不同的系数，然后加起来。

比如，如果有两个向量 v1 = (1, 0) 和 v2 = (0, 1)，
那么 2*v1 + 3*v2 = ?

你算一下。"
```

**步骤 3：记录轮数**

```python
轮数 = 0

while 学生未理解:
    轮数 += 1
    
    if 轮数 == 1:
        学生尝试回答
        
    if 回答错误:
        给提示，继续引导
        
    if 轮数 > 5:
        break  # 超过 5 轮，停止测试
```

**步骤 4：根据轮数分类**

| 轮数 | 吞吐率 | 推荐配置 |
|------|--------|---------|
| 1-2 轮 | 高 | `speed: sprint`（每轮 5-8 个概念） |
| 3-4 轮 | 中 | `speed: standard`（每轮 2-4 个概念） |
| 5+ 轮 | 低 | `speed: deep`（每轮 1-2 个概念） |

**步骤 5：写入配置**

```yaml
# learning-config.yaml
learning_pace:
  speed: standard  # 根据测试结果
  measured_throughput: 3  # 实测轮数
  measured_at: "2026-04-22"
```

## 常见陷阱

### 陷阱 1：相信学生的自我评估

**错误做法**：
```
导师："你了解向量吗？"
学生："了解。"
导师："好的，那我们直接开始线性组合。"
→ 学生其实只是听说过向量，并不真懂
```

**正确做法**：
```
导师："你了解向量吗？"
学生："了解。"
导师："那用你自己的话解释一下向量是什么？"
学生："呃……就是一个箭头？"
导师："能举个例子吗？"
学生："……"
→ 发现学生其实不懂，需要从头学
```

### 陷阱 2：验证问题太简单

**错误做法**：
```
导师："你知道什么是矩阵吗？"
学生："知道，就是一个数字的表格。"
导师："对，那我们继续。"
→ 学生只是记住了定义，并不理解矩阵的用途
```

**正确做法**：
```
导师："你知道什么是矩阵吗？"
学生："知道，就是一个数字的表格。"
导师："矩阵可以用来做什么？"
学生："呃……不太清楚。"
导师："好的，那我们先补充一下矩阵的应用场景。"
→ 发现学生只是半懂，需要加固
```

### 陷阱 3：跳过误概念拆除

**错误做法**：
```
导师："你了解导数吗？"
学生："了解，导数就是斜率。"
导师："好的，那我们继续学积分。"
→ 学生带着误概念继续学，后续概念全部悬空
```

**正确做法**：
```
导师："你了解导数吗？"
学生："了解，导数就是斜率。"
导师："那 y = x^2 在区间 [0, 2] 上的导数是多少？"
学生："呃……平均斜率？"
导师："我发现你可能把导数和斜率混淆了。导数是局部性质，斜率是全局性质。我们先澄清这个。"
→ 先拆除误概念，再继续
```

## 输出格式

验证完成后，必须写入 `student-profile.md`：

```markdown
---
created_at: 2026-04-22
last_updated: 2026-04-22
---

# 学生画像

## 先验知识地图

### 已验证的锚点（可直接使用）
- **向量**（2026-04-22）：能准确解释定义、举例、识别边界
- **函数**（2026-04-22）：理解函数的映射本质，能举例

### 需加固的概念（需补充后才能作为锚点）
- **矩阵**（2026-04-22）：知道定义，但不知道用途和应用场景
  - 加固策略：补充矩阵的几何意义（线性变换）

### 误概念（需拆除后重建）
- **导数**（2026-04-22）：混淆了导数和斜率，没理解导数是局部性质
  - 拆除策略：用反例暴露误概念（"区间上的导数"是无意义的）

### 未知概念（需从头学习）
- 特征值、特征向量
- 线性变换
- 基与维数

## 认知负荷特征

- **实测吞吐率**：3 轮/概念（中等）
- **推荐配置**：`speed: standard`
- **测试日期**：2026-04-22
- **测试概念**：线性组合（复杂度 = 3）
```

## 与 2.x 版本的区别

| 维度 | 2.x | 3.0 |
|------|-----|-----|
| 锚点探测 | 仅询问"你知道什么" | 询问 + 验证 + 分类（真懂/半懂/误概念/不懂） |
| 先验地图 | 学生自述 | 验证后的真实地图 |
| 认知负荷 | 凭经验估计 | 实测吞吐率 |
| 误概念处理 | 教学中发现时处理 | 入学时主动探测并拆除 |
